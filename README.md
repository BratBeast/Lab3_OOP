# Лабораторна робота №3: Multithreaded Algorithm Visualizer & Benchmarker

Проєкт реалізує паралельну мультипоточну версію алгоритмів та структур даних з графічним інтерфейсом користувача (Tkinter). Програма об'єднує вимоги Частини А (порівняння потоків) та Частини B (імітація, GUI, безпека потоків) Лабораторної роботи №3 з ООП.

## Особливості реалізації багатопоточності

### 1. Фонове виконання та потокобезпечний GUI
Для забезпечення чуйності інтерфейсу (щоб вікно не зависало під час сортування великих масивів), виконання алгоритмів винесено у фоновий потік (`threading.Thread`). Обмін даними між бекендом (алгоритмом) та фронтендом (Tkinter) відбувається через потокобезпечну чергу `queue.Queue`. Головний потік (UI) періодично опитує чергу і відмальовує лише найновіші зміни, пропускаючи застарілі кадри для оптимізації рендеру.

### 2. Паралельний алгоритм (Parallel Merge Sort)
Реалізовано багатопотокову версію сортування злиттям. Алгоритм рекурсивно розбиває масив на частини. Якщо кількість активних потоків не перевищує ліміт, заданий користувачем (1, 2, 4, 8, 16), для сортування кожної підмножини створюється окремий потік (`threading.Thread`). Коли ліміт вичерпується, алгоритм продовжує сортування послідовно в межах поточних потоків.

## Порівняння результатів (Бенчмаркінг)
Програма автоматично збирає метрики (час виконання, розмір масиву, кількість потоків) та експортує їх у файл `results.json` за допомогою патерну Builder.

**Аналіз паралельної vs послідовної версії:**
* **Накладні витрати (Overhead):** Для невеликих масивів (до 500 елементів) послідовні алгоритми (Quick Sort) або Timsort працюють швидше, оскільки створення та синхронізація нових потоків займає більше часу, ніж саме сортування.
* **Прискорення:** На масивах великого розміру (від 5000 елементів) `Parallel Merge Sort` з 4-8 потоками демонструє стабільніший час виконання.
* **GIL у Python:** Через Global Interpreter Lock (GIL) у CPython потоки виконуються псевдопаралельно (конкурентно). Реальне математичне прискорення досягається за рахунок того, що GUI-потік та потік обчислень розділені, що запобігає блокуванню процесу.

## Архітектура та Патерни Проєктування (GoF)
* **Singleton:** `SettingsManager` (Зберігає ліміт потоків).
* **Builder:** `JSONReportBuilder` (Експорт результатів у файл).
* **Strategy:** `SortStrategy` та спадкоємці (Динамічна заміна алгоритмів).
* **Command:** `RunAlgorithmCommand` (Інкапсулює запуск фонового потоку).
* **Memento:** Механізм "Відмінити (Undo)".
* **Facade:** `AlgorithmFacade` (Єдиний інтерфейс для GUI).
