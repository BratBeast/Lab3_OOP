# Algorithm Visualizer & Benchmarker

Проєкт для демонстрації роботи алгоритмів сортування та структур даних з вимірюванням їхньої продуктивності. Програма розроблена в рамках Лабораторної роботи №2 з дисципліни "Основи ООП".

## Функціонал та використання
* **Візуалізація:** Відображення роботи алгоритмів сортування у реальному часі за допомогою Tkinter (анімація процесу).
* **Бенчмаркінг:** Збір метрик часу виконання.
* **Інтерактивність:** Можливість скасувати останнє сортування (Undo) та повернути масив до початкового стану.
* **Експорт:** Автоматичне збереження результатів (розмір масиву, час виконання) у файл `results.json`.

### Як запустити
1. Встановіть Python 3.10+
2. Запустіть файл: `python src/main.py`
3. Оберіть алгоритм, тип вхідних даних та розмір масиву в графічному інтерфейсі.
4. Натисніть "Відсортувати" для запуску. Після завершення результати автоматично запишуться у файл `results.json`. Для скасування натисніть "Відмінити (Undo)".

## Архітектура та Патерни Проєктування (GoF)
Проєкт спроєктовано з використанням модульного підходу, де бекенд (логіка алгоритмів) повністю відділений від фронтенду (GUI). Для досягнення максимальної гнучкості реалізовано **10 патернів проєктування**.

### Породжувальні патерни (Creational)
1. **Singleton (Одинак)**
   * **Файл:** `src/algorithms/config.py` (`SettingsManager`)
   * **Проблема:** Необхідність єдиної точки доступу до налаштувань програми (розмір масиву, швидкість анімації) з різних модулів системи.
   * **Переваги:** Гарантує існування лише одного екземпляра конфігурації.
2. **Factory Method (Фабричний метод)**
   * **Файл:** `src/algorithms/factory.py` (`DataFactory`)
   * **Проблема:** Створення різних наборів вхідних даних (випадкові, вже відсортовані, зворотно відсортовані) без прив'язки клієнтського коду до конкретних реалізацій генераторів.
3. **Builder (Будівельник)**
   * **Файл:** `src/algorithms/report.py` (`ReportBuilder`, `JSONReportBuilder`, `ReportDirector`)
   * **Проблема:** Покрокове створення складного звіту про результати бенчмаркінгу та його збереження. Дозволяє легко додати експорт у CSV або XML у майбутньому (шляхом додавання нового конкретного будівельника).

### Структурні патерни (Structural)
4. **Facade (Фасад)**
   * **Файл:** `src/algorithms/facade.py` (`AlgorithmFacade`)
   * **Проблема:** Складна взаємодія між фабрикою даних, налаштуваннями та стратегіями алгоритмів. Фасад надає GUI єдиний простий метод `run_algorithm()`.
5. **Decorator (Декоратор)**
   * **Файл:** `src/algorithms/metrics.py` (`@timing_decorator`)
   * **Проблема:** Необхідність вимірювати час виконання алгоритмів. Декоратор дозволяє "обгорнути" будь-який алгоритм лічильником часу без зміни самого коду алгоритму.
6. **Adapter (Адаптер)**
   * **Файл:** `src/algorithms/sorting.py` (`BuiltInSortAdapter`)
   * **Проблема:** Інтеграція надшвидкої вбудованої функції Python `list.sort()` (Timsort) у наш стандартизований інтерфейс об'єктів для порівняння метрик.

### Поведінкові патерни (Behavioral)
7. **Strategy (Стратегія)**
   * **Файл:** `src/algorithms/base.py` та `sorting.py` (`SortStrategy` та його спадкоємці)
   * **Проблема:** Динамічна заміна алгоритмів сортування під час виконання програми. Користувач обирає алгоритм у GUI, а система підставляє відповідний клас, не змінюючи контекст.
8. **Template Method (Шаблонний метод)**
   * **Файл:** `src/algorithms/base.py` (`SortStrategy.execute`)
   * **Проблема:** Кожен алгоритм має виконати однакову послідовність дій (створити копію масиву, запустити сортування, повернути копію). Метод `execute` задає цей каркас, а `_sort` реалізується підкласами.
9. **Command (Команда)**
   * **Файл:** `src/gui/commands.py` (`RunAlgorithmCommand`)
   * **Проблема:** Жорстка прив'язка кнопок Tkinter до бізнес-логіки. Команда інкапсулює всі параметри запиту (алгоритм, тип даних, колбеки) в окремий об'єкт, забезпечуючи слабку зв'язність (loose coupling).
10. **Memento (Зберігач)**
    * **Файл:** `src/gui/memento.py` (`ArrayMemento`, `Caretaker`)
    * **Проблема:** Реалізація механізму "Відмінити (Undo)". Патерн зберігає попередній (невідсортований) стан масиву, дозволяючи відновити його без порушення інкапсуляції об'єктів бекенду.
